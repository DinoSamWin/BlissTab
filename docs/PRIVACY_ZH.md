# StartlyTab 隐私政策

最后更新：2026-04-27

本政策说明 StartlyTab 如何收集、使用、存储和共享数据，以满足 Chrome Web Store 对隐私披露的要求。

## 1. 我们会收集哪些数据

### 账号与登录数据

- 电子邮箱、账号 ID、昵称、头像
- 登录状态与必要的会话信息
- 后端返回的订阅状态、会员状态

### 用户保存的内容与设置

- 快捷入口的 URL、标题、分类、自定义名称
- 用户上传的网关 Logo 及其存储路径
- 主题、语言、搜索引擎、persona、自定义提示词

### 仅在用户主动操作时访问的浏览器数据

- 书签：仅当你主动使用书签导入功能时读取
- 当前标签页 URL、标题、favicon：仅当你主动用弹窗/侧栏添加当前页面时读取
- 站点标题和图标请求：当你保存某个网站为快捷入口时，StartlyTab 可能请求该页面或其图标地址以获取标题/图标

### 可选的“上下文感知模式”数据

仅当你在产品内明确同意后，StartlyTab 才会读取以下信号：

- 打开的标签页数量
- 是否有标签页正在播放音频
- 是否有下载任务正在进行
- 当前窗口状态
- 电池电量
- 你在当前页面上的空闲时长
- 以上仅作为聚合状态信号使用，而不是作为你所有打开标签页 URL 的完整列表使用

### 本地情绪与历史数据

- 你主动点击的情绪标签
- 本地保存的观点生成历史和时间戳
- 本地缓存的元数据

### 订阅与支持相关数据

- 订阅方案、兑换状态、交易参考 ID
- 用于支付门户或账单支持的邮箱
- 邮件验证、重置密码等事务邮件状态

## 2. 我们如何使用这些数据

我们仅将数据用于提供或改进 StartlyTab 的用户功能：

- 登录、鉴权、恢复同步后的工作区
- 保存、编辑、同步、导入、导出快捷入口与 Logo
- 基于你的语言、提示词、历史记录、情绪选择生成 AI 文案
- 如果你启用了“上下文感知模式”，AI 生成时还会使用可选的浏览器状态信号；这里设计上使用的是诸如标签页数量、是否有音频等汇总状态，而不是所有打开标签页的完整 URL
- 校验订阅、兑换码和会员权限
- 发送验证邮件、密码重置邮件，处理缓存和同步安全逻辑

## 3. 数据存储位置与保留期限

### 本地存储

StartlyTab 会在浏览器本地存储中保存应用状态、偏好设置、缓存、情绪记录和观点历史。Firebase 登录也可能使用 IndexedDB。

### 云端存储

同步后的用户数据、订阅状态、上传的 Logo、以及通过后端提交的反馈，可能存储在 Supabase。

### 云端保留

与账号关联的同步数据通常会在账号处于激活状态期间保留，直到你删除相关内容或向我们提出删除请求；如出于安全、反欺诈或法律合规需要，部分记录可能需要保留更长时间。

### 保留示例

- 情绪日志最多保留 30 天
- 本地观点历史会自动裁剪为近期记录
- 元数据缓存会在一段时间后自动过期

### 传输安全

网络传输预期通过 HTTPS/TLS 进行。云端访问受后端逻辑和服务商安全机制控制。

## 4. 数据会共享给谁

我们不会出售你的个人数据，也不会将其用于个性化广告、征信判断或数据经纪业务。

### 共享对象与用途

- **Google / Firebase**：Google 登录、Firebase 身份验证及相关账号会话处理
- **Supabase**：云同步、数据库、Logo 存储、后端函数
- **AI 服务商**：如 DeepSeek、SiliconFlow、ZhipuAI（按实际配置），用于在你请求生成时处理提示词和你选择启用的上下文摘要信号
- **Creem**：支付结账、订阅门户、账单相关查询
- **Resend**：发送验证码、重置密码等事务邮件
- **你保存的网站或其图标地址**：在你添加快捷入口时，请求页面标题或图标

## 5. 你的选择权

- 你可以不启用“上下文感知模式”，以基础模式使用 StartlyTab
- 你可以通过清理 StartlyTab 在浏览器中的本地存储，或卸载扩展，删除当前设备上的本地数据；如产品设置中提供了重置入口，也可以使用该入口
- 如需删除云端同步数据，请联系 `support@startlytab.com`；我们会据此删除无需因安全、反欺诈或法律合规继续保留的账号关联同步数据

## 6. Chrome Limited Use 声明

The use of information received from Google APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.
